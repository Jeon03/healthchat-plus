1. 기본 정보 & 핵심 가치

개발자: 전여욱

사용자의 행동과 감정을 이해하고, 과학적 근거 기반으로 더 건강한 방향을 제시하는 개발자

상세 소개

저는 헬스케어 서비스에서 가장 중요한 가치는 ‘사용자 경험’과 ‘근거 기반의 신뢰성’이라고 생각합니다.

HealthChat+는 사용자가 자연어로 입력한 식단·운동·감정 데이터를 AI가 분석하고, 국제 보건 기구(WHO), 질병관리청(KDCA), 한국인 영양섭취기준(KDRI) 등 공신력 있는 기준을 참고해 초개인화 피드백을 제공하는 것을 목표로 개발되었습니다.

핵심 차별점은 단순 요약이 아닌, “왜 이런 피드백을 주는가?”에 대한 근거/설명을 함께 제공하는 것입니다.

2-1. 기획 내역 (개발 목표)

HealthChat+는 기존 건강 관리 서비스가 가진 기록의 번거로움과 AI 피드백에 대한 신뢰 부족 문제를 해결하기 위해 기획되었습니다.

  • 자연어 기반 기록: 일상 문장 한 줄로 식단·운동·감정 기록
  • 근거 기반 AI 피드백: WHO·KDCA·KDRI 등 기준과 함께 설명 제공
  • 초개인화 건강 코칭: 목표·신체정보·수면·알레르기 반영
  • 하루 단위 통합 관리: 일일 로그(Daily Log) 중심으로 관리

2-2. 개발 기능

🧠 자연어 분석 및 라우팅

  • 사용자 입력 문장을 식단·운동·감정 영역으로 자동 분리
  • Gemini API 기반 구조화(JSON) 처리
  • 불완전 입력에 대한 fallback 로직

🍽 식단 · 🏃 운동 · ❤️ 감정 분석

  • 식단: 음식명/섭취 맥락 분석
  • 운동: 종류/시간 기반 활동 분석
  • 감정: 감정 상태/톤 분석

📒 일일 로그(Daily Log) 통합

  • 식단·운동·감정 데이터를 하루 단위로 통합 저장
  • 날짜별 조회 및 수정 기능
  • 대시보드용 집계 데이터 생성

🤖 AI Health Coach

  • Daily Log 기반 종합 건강 요약 생성
  • RAG DB 기반 근거/참고문헌 제공
  • 사용자 목표에 맞춘 개선 방향 제안

👤 사용자 프로필 관리

  • 신체 정보, 목표, 수면, 알레르기 설정
  • 프로필 기반 개인화 분석 로직 적용

복합 입력 예시 (아침/점심/저녁 포함)

“아침에 그릭요거트랑 바나나 먹었고, 점심은 김밥 1줄 먹었어. 저녁엔 삼겹살을 좀 많이 먹었는데 오늘은 스트레스가 있어서 20분 산책했어.”

3. 프로젝트 상세: 시스템 & 기능

문제 정의 (Problem Statement)

대부분의 건강 관리 앱이 단순 기록에 집중하고, 사용자 맥락(목표/수면/알레르기/감정)을 반영한 “신뢰도 높은 피드백” 제공에는 한계가 있다는 문제에서 출발했습니다.

아키텍처 및 설계 (Architecture & Design)

  • Routing Service가 자연어 입력을 식단·운동·감정 분석 서비스로 분기
  • 각 분석 결과는 Daily Meal / Daily Exercise / Daily Emotion으로 저장
  • 일일 로그(Daily Log)로 통합 후 AI Health Coach가 피드백 생성
  • WHO·KDRI 등 문헌은 RAG DB를 통해 근거 기반 참조

핵심 기능 (Core Features)

3.1. 자연어 기반 식단·운동·감정 통합 분석

사용자의 일상 문장에서 AI가 핵심 건강 데이터를 추출해 DB에 구조화합니다.

  • AI 추출 데이터: 식단(음식명/영양소), 운동(종류/강도/시간), 감정(카테고리/강도)

기술 적용 예시

“오늘 우울해서 초콜릿 먹고 20분 산책했어” → 감정: 우울 / 음식: 초콜릿 / 운동: 산책 20분

3.2. 공신력 기반 AI 초개인화 건강 요약

AI 피드백을 “근거 기반”으로 구성해 신뢰도를 높였습니다. (분석 결과 + 근거/설명 + 실행 가능한 제안)

  • 주요 참조 기준: WHO 운동 권장, KDCA 가이드, KDRI 등

피드백 예시

“WHO 성인 권장 운동량(주 150~300분) 대비 최근 평균 운동 시간이 부족한 편입니다.”

신뢰성 설계

근거(가이드라인/논문/기관 자료)를 “참고문헌” 형태로 노출해, AI 환각 리스크를 줄이고 사용자가 판단할 수 있게 했습니다.

3.3. 감정 기반 건강 트렌드 분석

감정 데이터와 건강 데이터를 함께 보며 생활 패턴을 스스로 인지하도록 돕습니다.

  • 스트레스/우울 등 감정 변화 ↔ 섭취/운동 변화 상관관계 탐색
  • 대시보드에서 추세(예: 우울 점수 vs 운동 시간) 시각화

3.4. 개인 프로필 기반 맞춤 분석

목표/알레르기/복용약 등 사용자 맥락을 반영해 더 현실적인 코칭을 제공합니다.

  • 목표 체중/목표에 따른 칼로리 밸런스 판단
  • 알레르기/복용약 기반 주의 식품/습관 안내
  • 수면 데이터와 연계한 회복/컨디션 코멘트

트러블슈팅 (Troubleshooting)

1) LLM 호출 실패 및 비정형 응답 문제

LLM 호출 시 JSON 형식이 깨지거나 불완전한 응답이 발생했습니다. 프롬프트를 역할·출력 형식 중심으로 단순화하고, fallback 처리 및 원문 로그를 추가해 안정성을 확보했습니다.

2) LLM 응답 속도 지연 및 비용 문제

단일 요청에 과도한 분석을 포함해 응답 시간이 길어지는 문제가 있었습니다. 이를 해결하기 위해 식단·운동·감정 라우팅 구조로 분리하고, 프롬프트를 최소화하여 속도와 비용을 함께 최적화했습니다.

3) 프롬프트 설계에 따른 분석 일관성 문제

프롬프트 표현에 따라 분석 결과가 달라지는 문제가 있어, 출력 필드를 명시한 구조화 프롬프트(JSON 스키마)를 적용했습니다. 결과 일관성이 향상되고 후처리 로직이 단순화되었습니다.
