상세 소개

저는 헬스케어 서비스에서 가장 중요한 가치는 ‘사용자 경험’과 ‘근거 기반의 신뢰성’이라고 생각합니다.

HealthChat+는 사용자가 자연어로 입력한 식단·운동·감정 데이터를 AI가 분석하고, 국제 보건 기구(WHO), 식품영양학 기준, 질병관리청 자료 등의 정보를 근거로 한 초개인화 피드백을 제공하는 것을 목표로 개발되었습니다.

핵심 차별점은 단순 요약이 아닌, “왜 이런 피드백을 주는가?”에 대한 과학적 설명을 함께 제공하는 것입니다. 이 프로젝트를 통해 AI 분석 기술, 건강 데이터 구조화, 신뢰성 있는 피드백 엔진 설계 등 풀스택 관점의 다양한 개발 경험을 쌓았습니다.

3. 프로젝트 상세: 시스템 & 기능

문제 정의 (Problem Statement)

대부분의 건강 관리 앱이 단순 기록에만 집중하고, 공신력 있는 근거 기반의 “신뢰도 높은 피드백” 제공에 한계가 있다는 문제점에서 프로젝트를 시작했습니다.

아키텍처 및 설계 (Architecture & Design)

  • 데이터 플로우: 자연어 입력 → Gemini Function Calling → 음식·운동·감정 분석 및 구조화
  • AI 피드백: WHO, 질병관리청, 영양학 논문 등의 자료를 참고한 규칙 기반 가이드 포함.
  • 개인화: 사용자 프로필(키, 몸무게, 목표, 수면, 알레르기) 기반 초개인화 분석.
  • 구조: Spring Boot REST API (Backend) + React/Tailwind CSS (Frontend)
  • 배포: AWS EC2 기반 Docker 컨테이너 배포

핵심 기능 (Core Features)

3.1. 자연어 기반 식단·운동·감정 통합 분석

사용자의 일상적인 자연어 입력에서 AI가 핵심 건강 데이터를 자동 추출하고 DB에 구조화합니다.

  • AI 추출 데이터: 식단 (음식명, 칼로리, 영양소), 운동 (종류, 강도·시간), 감정 (톤, 키워드)

기술 적용 예시:

“오늘 우울해서 초콜릿 먹고 20분 산책했어” → Gemini Function Calling을 통해 감정: 우울, 음식: 초콜릿, 운동: 산책 20분으로 정형화하여 저장.

3.2. 공신력 기반 AI 초개인화 건강 요약

AI 피드백은 공신력 있는 기관의 기준을 근거로 제시되어 신뢰도를 극대화합니다. AI 분석 결과 + 근거 설명 구성.

  • 주요 참조 기준: WHO 권장량/운동 기준, 질병관리청(KDCA) 가이드, 한국인 영양섭취기준(KDRI).

피드백 예시:

“WHO 기준 성인의 하루 권장 운동량(150~300분) 대비 최근 7일 평균 운동 시간이 부족한 편입니다.”

3.3. 감정 기반 건강 트렌드 분석

감정 데이터와 건강 데이터의 결합 분석을 통해 사용자 스스로 자신의 생활 패턴을 진단할 수 있도록 지원합니다.

  • 분석 항목: 감정 변화가 식습관/운동 패턴에 미치는 상관관계 분석.
  • 트렌드 시각화: 스트레스-탄수화물 섭취량, 우울 점수-운동 시간 등 상관관계 그래프 제공.

3.4. 개인 프로필 기반 맞춤 분석

사용자 프로필 설정 정보(목표, 알레르기, 약 복용 등)를 반영하여 정교하고 맞춤화된 코칭을 제공합니다.

  • 맞춤 분석: 목표 체중 기반 칼로리 초과 여부 판단, 알레르기 위험 감지, 약 복용에 따른 식단 조정 등.
